# MVP Product Requirements

## Outcome

用户输入公开的 Steam `/profiles/<steamid64>` URL、Steam ID64 或 Dota Account ID 后，可以查看最近 100 场已导入比赛、英雄使用统计和比赛详情，并查询当前版本英雄、物品与地图百科。Vanity URL 需要额外 Steam API 能力，不属于首版。

## P0 journeys

1. 账号查询 → 同步状态 → 玩家概览。
2. 玩家概览 → 英雄使用列表 → 玩家英雄详情。
3. 玩家概览 → 最近比赛 → 单场比赛详情。
4. 比赛中的英雄/物品 → 百科详情。
5. 导航 → 当前地图 → 版本化地点说明。

## P0 pages

- 账号查询与同步页。
- 玩家概览页。
- 玩家英雄详情页。
- 比赛详情页。
- 英雄列表与详情页。
- 物品列表与详情页。
- 当前地图百科页。
- 数据状态页：展示整体/provider 状态、检查时间、来源和最新数据水位；未定义口径前不展示伪造的全局成功率。

## Account statistics

- 已导入比赛数、胜负与胜率。
- 英雄使用场次、使用占比、胜率。
- 总量 KDA 与场均 K/D/A。
- 数据存在时显示 GPM、XPM、补刀和英雄伤害。
- 完整比赛详情优先展示十人阵容、等级、GPM/XPM、补刀/反补、伤害、最终装备与技能升级顺序。
- 购买/出售时间线只有在上游或 replay 提供真实事件时展示；不可用状态不得退化为空时间线。
- 最近 20/50/100 场；不足时显示实际样本量。
- 所有列表稳定排序：`start_time DESC, match_id DESC`。

## Explicit non-goals

- 私密比赛历史解锁或绕过。
- 完整职业生涯承诺。
- 精确购买时间、技能加点时间线和每分钟曲线。
- 眼位、死亡、移动、团战热图和推荐路线。
- 修正后的克制/配合模型。
- React Native、GraphQL、公开开发者 API 和自研 replay parser。

## Acceptance

- 一个公开账号可以幂等同步最近 100 场比赛。
- 玩家总场次、英雄汇总和比赛列表可逐场对账。
- 同一比赛重复导入不重复计数。
- 私密、部分数据、限流、上游异常和失败有独立状态。
- 当前版本英雄、物品可查询；地图只有在来源、Patch 和 geometry 均已验证时才返回，否则明确返回 `MAP_UNAVAILABLE`。
- 地图快照必须可追溯到指定 Steam App 570 build/depot/resource hash 与提取器版本；不使用官方 minimap 纹理，不手写坐标。未取得合法内容快照与用途许可时，生产继续显示不可用状态。
- 物品定义必须区分普通物品、配方、中立物品与中立强化项；定义存在但当前可购买性未验证时明确显示 `unverified`。
- 每个统计值带样本范围、覆盖率、来源、版本和更新时间。
- 冻结黄金数据集可重复计算出完全相同的结果。
