## Design Context

### Users

Dota 2 玩家会在桌面与竖屏手机上高频查看自己的公开比赛、英雄使用、装备与地图资料。他们需要快速扫描结果、比较趋势、切换统计窗口，并从账号概览下钻到英雄、比赛和物品，而不是阅读大段解释。

### Brand Personality

克制、锐利、可信。

界面应像一套经过长期打磨的竞技数据终端：信息密度高但不拥挤，状态明确但不喧闹，视觉张力来自真实英雄/物品素材与数据变化，而不是装饰性光效。

### Aesthetic Direction

- 暗色灰黑背景，带极轻微冷色倾向；不使用纯黑或纯白。
- 深浅表面主要通过明度、细边框和留白区分，不依赖通用阴影或玻璃拟态。
- 英雄头像、物品图标与地图内容承担主要鲜艳色彩。
- 绿色表示正向、胜利或上升，红色表示负向、失败或下降；颜色必须同时搭配符号、文字或形状，不能成为唯一信息通道。
- 数据模块具有明确职责：账号概览、英雄、比赛、物品、地图分别组织；模块内部可通过 segmented control、tab 或时间窗口按钮切换数据。
- 桌面采用不对称高密度仪表盘；竖屏重新排列优先级，先显示账号结论与关键趋势，再显示可横向滚动或折叠的明细，不简单压缩桌面网格。
- 动效只用于模块进入、筛选切换与数据状态反馈；遵守 reduced-motion，避免弹跳、霓虹光晕和持续装饰动画。

### Typography

品牌词为“克制、锐利、可信”。已主动排除常见反射选择 Inter、Space Grotesk 与 IBM Plex Sans。

- Display / numeric: Saira Condensed，用于拉开标题、关键数字和紧凑数据标签的节奏。
- UI / Chinese body: Noto Sans SC，用于中文正文、控件和表格，优先保证高密度下的清晰度。
- 不用等宽字体把“技术感”当作捷径；数字通过 tabular numerals 与对齐实现可比性。

### Design Principles

1. 先结论后明细：每个模块先回答“发生了什么”，再提供切换和下钻。
2. 让真实内容发光：英雄与物品素材鲜艳，框架本身保持低饱和、低噪声。
3. 趋势有双重编码：颜色之外必须有方向箭头、正负号、标签或形状。
4. 竖屏重排而非缩放：保留全部关键功能，减少同时竞争注意力的列数。
5. 数据可信度可见：样本量、覆盖率、来源、窗口和更新时间随统计一起出现。
6. 状态完整：Loading、Empty、Private、Partial、Rate limited、Unavailable 与 Failed 都有独立且可行动的表现。
